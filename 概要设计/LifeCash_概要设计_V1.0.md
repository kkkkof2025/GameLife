## 一、系统架构

### 1.1 整体架构
`
┌─────────────────────────────────────────────────────────┐
│                    前端 (HTML5 Game)                     │
├─────────────────────────────────────────────────────────┤
│  渲染层 (Canvas/DOM) │ UI层 │ 逻辑层 │ 网络层           │
├─────────────────────────────────────────────────────────┤
│                    后端 (Node.js Server)                 │
├─────────────────────────────────────────────────────────┤
│           WebSocket Server │ Room Manager │ DB           │
└─────────────────────────────────────────────────────────┘
`

### 1.2 技术选型
| 层级 | 技术 | 说明 |
|------|------|------|
| 前端核心 | HTML5 + Canvas | 游戏渲染 |
| 前端逻辑 | 原生JavaScript | 游戏引擎 |
| 通信 | Socket.io | 多人实时同步 |
| 后端 | Node.js + Express | API服务 |
| 数据库 | SQLite | 本地存储 |
| WebSocket | Socket.io | 实时通信 |

## 二、核心模块设计

### 2.1 游戏引擎模块 (GameEngine)
**职责**: 游戏主循环、状态管理、时间推进
`
GameEngine
├── init()              // 初始化游戏
├── start()             // 开始游戏
├── pause()             // 暂停游戏
├── update(deltaTime)   // 更新游戏状态
├── render()            // 渲染画面
└── dispose()           // 销毁游戏
`

### 2.2 角色模块 (Player)
**职责**: 角色属性、状态、成长
`
Player
├── 属性: 健康度,幸福度,金钱,资产,经验
├── 方法: 工作(),投资(),休息(),社交()
├── 状态机: 正常/破产/自由/富人
└── 成长: 技能树,等级,属性提升
`

### 2.3 职业模块 (Career)
**职责**: 职业定义、属性差异
`
Career
├── 工薪族 | 自由职业者 | 创业者 | 公务员 | 投资人 | 教师
├── 扩展: 医生/律师/艺术家/运动员/网红
└── 特性: 初始属性,技能,成长曲线
`

### 2.4 地图模块 (GameMap)
**职责**: 地图节点、路径、事件触发
`
GameMap
├── 节点: 工作/机会/市场/休息/社交/健康/随机
├── 移动: 骰子/手动选择
└── 事件: 触发节点对应事件
`

### 2.5 事件系统 (EventSystem)
**职责**: 事件定义、触发、执行
`
EventSystem
├── 事件库: 机会/风险/社交/惊喜
├── 触发器: 概率/条件/随机
└── 执行器: 属性变化/状态更新
`

### 2.6 投资系统 (InvestmentSystem)
**职责**: 投资品种、收益计算、风险管理
`
InvestmentSystem
├── 品种: 股票/基金/房产/债券/黄金/创业
├── 计算: 收益/风险/流动性
└── 策略: 被动收入,FIRE指标
`

### 2.7 多人系统 (MultiplayerSystem)
**职责**: 房间管理、状态同步、排行榜
`
MultiplayerSystem
├── 房间: 创建/加入/离开
├── 同步: 状态心跳,操作广播
└── 排行: 实时排名,历史记录
`

### 2.8 UI模块 (UIManager)
**职责**: 界面渲染、用户交互
`
UIManager
├── 主界面: 状态栏/地图/操作按钮
├── 职业选择: 职业卡片/属性预览
├── 投资面板: 品种列表/交易界面
└── 结果报告: 统计数据/成就展示
`

## 四、游戏模式设计

### 4.1 畅玩模式（无限模式）
- 无时间限制
- 自由探索
- 触发条件：
  - 破产 → 进入"东山再起"阶段
  - 财务自由 → 进入"富人模式"
- 富人模式特性：
  - 超级项目解锁
  - 奢侈品系统
  - 影响力系统

### 4.2 限时模式（竞技模式）
- 时间设置：30min/1h/2h/自定义
- 多人房间：2-6人
- 实时排名
- 最终报告生成

## 六、网络接口设计（多人模式）

### 6.1 WebSocket事件
| 事件名 | 方向 | 说明 |
|--------|------|------|
| join_room | C→S | 加入房间 |
| leave_room | C→S | 离开房间 |
| player_action | C→S | 玩家操作 |
| game_state | S→C | 游戏状态同步 |
| chat_message | C↔S | 聊天消息 |

### 6.2 REST API
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/rooms | 获取房间列表 |
| POST | /api/rooms | 创建房间 |
| GET | /api/rooms/:id | 获取房间详情 |
| DELETE | /api/rooms/:id | 删除房间 |
| GET | /api/leaderboard | 排行榜 |
